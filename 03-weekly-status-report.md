# 3. Weekly Status Report Template

Goal: the sponsor should be able to read this in 90 seconds and never be surprised by a launch delay. If something bad is happening, it should show up here *before* the sponsor hears it from someone else.

```markdown
# Weekly Status Report — [Project Name]
Week of: [date] · Report #: [n] · Prepared by: [name]

## 🚦 Overall Status: 🟢 On Track / 🟡 At Risk / 🔴 Blocked

One sentence explaining the color. Example: "🟡 At Risk — data migration dry run surfaced a 4% SKU mismatch, root cause identified, fix in progress."

## This Week's Progress
- [Completed item 1]
- [Completed item 2]
- [Completed item 3]

## Next Week's Plan
- [Planned item 1]
- [Planned item 2]

## Metrics vs Plan
| Metric | Target | Actual | Status |
|---|---|---|---|
| % of milestones on schedule | | | |
| Budget spent vs planned | | | |
| Open critical bugs | | | |

## Risks & Blockers (escalate here, don't bury them)
| Item | Impact if unresolved | Owner | Needed by |
|---|---|---|---|
| | | | |

## Decisions Needed From Sponsor
- [ ] [Decision 1 — deadline]
- [ ] [Decision 2 — deadline]

## Changes to Scope/Timeline/Budget Since Last Report
[None] or [describe + link to change request]
```

## 3.1 Rules for Using This Template

1. **Send it the same day/time every week.** Consistency builds trust more than content does.
2. **Never let "Overall Status" stay green while a risk score is ≥15 in the risk matrix.** If it's red in the matrix, it can't be green in the report.
3. **"Decisions Needed" must have a deadline.** An open-ended ask gets deprioritized by sponsors every time.
4. **Keep it to one page.** If it's longer, move detail into an appendix link, not the report body.
